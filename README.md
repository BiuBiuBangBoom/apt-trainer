# APT Trainer — 行测心算训练助手

行政职业能力测验（行测）速算练习工具，交互式终端应用。覆盖 10 种速算题型，支持 3 种练习模式，配备基于规则分析 + DeepSeek 大模型的智能薄弱点分析系统。

## 快速开始

```bash
conda activate python313
python main.py            # 正常运行
python main.py -n 20      # 限制 20 题
python main.py -t 30      # 每题限时 30 秒
```

无外部依赖，仅需 Python 3.13 标准库。

## 练习模式（3 种）

### 练习模式 `[1]`

每道题作答后立即显示正确答案，适合初学者建立手感。

### 练习测验模式 `[2]`

每题即时判断对错并显示正确率标记，退出时展示完整答题详情和成绩汇总，适合日常训练。

### 考试模式 `[3]`

全程不反馈对错，退出时统一展示成绩。模拟真实考试心理状态。

| # | 题型 | 示例 | 考察能力 |
|---|------|------|----------|
| 1 | 两位数乘一位数 | 74 × 6 | 心算乘法、进位处理 |
| 2 | 一位数加一位数 | 7 + 5 | 基础加法、进位敏感度 |
| 3 | 一位数乘一位数 | 8 × 7 | 乘法口诀熟练度 |

## 历史统计 `[0]`

查看所有历史练习数据，按题型汇总：

- 练习次数
- 正确率（绿色 ≥85%，黄色 ≥70%，红色 <70%）
- 平均耗时

支持一键清空全部记录。

## 错题回顾 `[R]`

从历史记录中提取所有答错的题目，按题型分组展示。可选择：

- 回顾特定题型的错题
- 一次性回顾全部错题

错题原样重现，用练习测验模式重新作答。适合考前查漏补缺。

## 智能薄弱分析 `[W]`

这是本工具的核心差异化功能。与传统的"按题型统计正确率"不同，智能薄弱分析会深入**数字级别**挖掘你的真正弱点。

### 它做了什么

传统的薄弱训练只能告诉你"你两位数乘一位数比较差"，但为什么差？是对所有乘法都慢，还是特定数字组合容易出错？

智能薄弱分析通过两层分析回答这个问题：

**第一层 — 规则分析（本地，免费）**

系统对每道题的内部结构提取细粒度特征。举个例子，`74 × 6` 这道题会被拆解为：

| 特征维度 | 值 |
|---------|-----|
| 乘数 | 6 |
| 进位次数 | 2 |
| 十位数 | 7 |
| 个位数 | 4 |
| 结果位数 | 3 |

然后统计每个特征值的历史表现（正确率、平均耗时），与题型基线对比。当某个特征值的正确率显著低于该题型均值（差距 >15%）或耗时显著偏高（>1.5 倍），就会被标记为薄弱点。

每种题型有自己的特征维度体系，10 种题型共约 30 个特征维度，覆盖了数字大小、进位/退位次数、结果量级、数值接近程度等计算难度因素。

同时进行跨题型汇总——追踪数字 0-9 在所有题型中的表现，发现类似"你对含数字 7 的计算整体偏慢"这样的跨题型规律。

**第二层 — DeepSeek 大模型分析（可选，需配置 API key）**

将规则分析结果和抽样原始数据发送给 DeepSeek，让 AI 做三件事：

1. **验证规则发现**：判断规则标记的薄弱点是否合理（confirmed / questionable / false_positive）
2. **补充遗漏规律**：发现规则分析难以捕捉的组合模式，如"个位数为 4 且乘数 ≥6 时错误率偏高，可能与双重大数进位的心算负担叠加有关"
3. **给出训练建议**：根据薄弱点优先级推荐训练顺序

### 如何配置大模型分析

在项目根目录创建 `settings.json`：

```json
{
  "api_key": "sk-你的deepseek-api-key"
}
```

不配置 API key 时，仅进行规则分析。文件已加入 `.gitignore`，不会被提交。

### 分析报告

报告分四个区域：

```
智能薄弱分析报告
─────────────────
规则发现 — 题型专项弱点     ← 基于统计的确定性发现，编号可选
  [1] 两位数乘一位数  进位次数=2
      正确率: 40.0%  耗时: 6.2s  样本: 5题

跨题型弱点                ← 跨题型的数字敏感度，仅展示
  数字 7 敏感度低
      正确率: 45.0%  涉及: 一位数加一位数, 两位乘一位数

AI 发现 — 可训练项目       ← LLM 发现的组合规律，编号可选
  [4] 两位数乘一位数  个位数=4, 乘数=6
      个位数为4且乘数≥6时错误率偏高

AI 深度分析               ← LLM 的验证意见和技术解释
  训练建议                 ← LLM 给出的优先级建议
```

### 针对性训练

选择规则发现或 AI 发现中的任意一项，系统会自动生成 **20 道符合该薄弱特征约束的题目**。

例如选择 `进位次数=2`，系统会专门生成双进位乘法题（如 74×6、57×8、89×4），而不是普通的随机乘法。选择 AI 发现的 `个位数=4, 乘数=6`，则生成同时满足两个约束的题目。

训练采用练习测验模式，做完后显示统计，并自动保存记录（持续积累数据，让未来的分析更精准）。

### 数据积累

分析需要足够的样本（每个特征值至少 3 条记录）才会触发标记。建议先正常练习 2-3 周积累数据，再使用此功能。随着数据增长，分析的精度和覆盖范围会持续提升。

## 命令行参数

| 参数 | 说明 |
|------|------|
| `-n N` | 限制每次练习的题目数量 |
| `-t N` | 每题限时 N 秒（暂未强制中断，仅作提示） |

## 架构

双层策略模式：

- **第一层 — ExamStrategy**：控制答题流程和反馈方式（练习/测验/考试）
- **第二层 — QuestionType**：纯函数式出题和判题，无副作用

`Question.generate_question()` 返回自包含的 `Question` dataclass（题目文本 + 答案 + 操作数元数据），`Session` 管理题目生成和答题记录，`DataStore` 以 TSV 格式持久化。

分析系统：

- **analyzer.py** — 规则分析引擎：特征提取 → 统计 → 对比基线 → 标记异常 → 跨题型汇总
- **llm_analyzer.py** — DeepSeek 深度分析：发送规则发现 + 数据抽样 → AI 验证 + 补充 + 建议
- **mode.py** — 特征提取 / 针对性出题：每道题可被拆解为特征向量，也可按特征约束生成
